# Teste para candidatos à vaga de Desenvolvedor Back-end

Olá caro candidato, nesse teste analisaremos seu conhecimento geral e inclusive velocidade de desenvolvimento. Abaixo explicaremos tudo o que será necessário.

## Responsabilidades

 - Integrar nosso time e liderar a arquitetura e desenvolvimento de soluções escaláveis, resilientes e seguras. 
 - Atuar como Líder Técnico, orientando e apoiando as equipes de desenvolvimento em desafios técnicos e garantindo a qualidade do código através de revisões
 - Garantir a integridade, performance e segurança das aplicações, resolvendo problemas complexos de software de maneira eficiente e simples
 - Mentorar outros desenvolvedores, proporcionando suporte técnico e orientação para melhorar a performance das equipes.

## Pré-Requisitos:

### Qualificações:

 - Experiência sólida como desenvolvedor full stack.
 - Proficiente em bancos de dados relacionais.
 - Experiência com Redis para otimização de cache e desempenho.
 - Familiaridade com serviços da AWS, incluindo EC2, S3, SES e RDS.

### Diferenciais:

- Experiência em outras linguagens de programação.
- Conhecimento em práticas de DevOps e CI/CD.
- Experiência com metodologias ágeis e ferramentas de gerenciamento de projetos, como Jira.

## Instruções

O desafio consiste em implementar uma aplicação web utilizando o framework PHP Laravel, um banco de dados relacional (Mysql ou Postgres), que terá como finalidade a matrícula de alunos de alunos em cursos através de um painel administrativo.

Sua aplicação deve possuir:

- CRUD de cursos:
  - Criar, editar, excluir e listar vagas.
  - O curso pode on-line ou presencial.
- CRUD de alunos:
  - Criar, editar, excluir e listar alunos.
- Um aluno pode se matricular em um ou mais cursos.
- Os cursos deverão ter uma data máxima para receber novas matrículas.
- Os cursos poderão ter uma quantidade máxima de matrículas.
- Cada CRUD:
  - Deve ser filtrável por qualquer campo, e possuir paginação de 15 itens.
  - Deve possuir formulários para criação e atualização de seus itens.
  - Deve permitir a deleção de qualquer item de sua lista.
  - Implementar validações de campos obrigatórios e tipos de dados.
- Testes unitários e de unidade.

## Banco de dados

- O banco de dados deve ser criado utilizando Migrations do framework Laravel, e também utilizar Seeds e Factorys para popular as informações no banco de dados.

## Tecnologias a serem utilizadas

Devem ser utilizadas as seguintes tecnologias:

- HTML
- CSS
- Javascript
- Framework Laravel (PHP)
- Docker (construção do ambiente de desenvolvimento)
- Mysql, Postgres

## Entrega

- Para iniciar o teste, siga atentamente as instruções do teste;
- Crie um repositório público na sua conta do GitHub;
- Faça o arquivo readme.md com as informações necessárias para executar o seu teste (comandos, migrations, seeds, etc);
- Depois de finalizado, envie-nos o link do seu teste;

## Bônus

- API Rest JSON para todos os CRUDS listados acima.
- Permitir deleção em massa de itens nos CRUDs.
- Permitir que o usuário mude o número de itens por página.
- Implementar autenticação de usuário na aplicação.

## O que iremos analisar

- Organização do código;
- Aplicação de design patterns;
- Aplicação de testes;
- Separação de módulos e componentes;
- Legibilidade;
- Criação do ambiente com Docker.
